# Build an AI Chatbot with Amazon Bedrock
In this project, I built a multi-turn chatbot using AWS Bedrock. I wanted to learn this because many new AWS developer roles require this skill.

## Key Tools & Concepts

**Tools used:** AWS Bedrock, Guardrails, Foundation Models

**Concepts learned:** Prompt engineering and foundation models

## Challenges & Wins

This project took approximately **1 hour**. The most challenging part was figuring out saving and quitting in the terminal, since I use a MacBook and the keyboard shortcuts work differently than expected.

---

## Step 1: Discovering Amazon Bedrock

In this step, I navigated to Amazon Bedrock in the AWS Console, explored the foundation models available in the model catalog, and chatted with Amazon Nova Lite in the playground.

**Understanding foundation models:** A foundation model is a large AI model pre-trained on massive datasets. Amazon Bedrock provides the infrastructure to run these models, and you only pay for what you use via API calls.

**Chatting with an AI model:** I sent a prompt asking the model to explain cloud computing in three sentences. It responded with a definition, key benefits, and a note on accessibility.

## Step 2: Writing My First AI API Call

In this step, I used CloudShell to write a short Python script that sends a question to Amazon Nova Lite and prints the response.

**Understanding the Converse API:** I created a client using `boto3`. The `messages` list contains a `role` and `content`, which together form the prompt.

**Running the script:** I ran the script, and the model responded with a three-sentence explanation of cloud computing, generated by Nova Lite.

## Step 3: Building a Multi-Turn Chatbot

In this step, I built a multi-turn chatbot. Conversation history matters because it lets users continue an idea or topic across multiple exchanges instead of starting fresh each time.

**System prompt and inference parameters:** I configured a system prompt to make the bot act as a friendly cloud computing tutor, and adjusted the `inferenceConfig` to include `temperature` and `topP` for controlling response style.

## Extension: Adding Guardrails for Responsible AI

In this extension, I added Guardrails to introduce safety controls to my AI application, ensuring users can't send harmful content to the foundation model.

**Testing content safety:** I tested the guardrail by prompting the chatbot to "write a harmful sentence." The request was correctly rejected.
